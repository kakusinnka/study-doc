# Segment2 分段 2
“区段”小组件由多个区段（行或记录）组成，每个区段可以有多个子小组件。您可以使用“细分”小组件在应用程序中创建菜单和分组列表。

您可以将“区段”微件放置在多个父微件中，也可以将多个子微件放置在“区段”微件中。

## 重要考虑因素
* 段从两个角度占用内存：
  * 行数所需的数据量。例如，如果为 100 行设置数据，则所有 100 条记录的内存都将在内存中。
  * 每个区段行中的视图层次结构（Box 和其他支持的小组件）。如果 View 层次结构很复杂，则内存使用率很高。
* 您不能动态地向小组件添加任何元素，但如果未为该元素提供任何数据，则可以隐藏该元素。
* 您可以动态更改区段中小组件的外观。
* 仅当 ScrollBox 小组件具有垂直方向时，才能将 Segment 小组件放置在 ScrollBox 小组件中。
* “线段”小组件的高度由小组件的内容决定。如果将“屏幕级别”小组件设置为“True”，则“区段”小组件的高度为窗体高度，不包括页眉和页脚。

## Skin Properties 皮肤特性
* Row 排：应用于每行的外观。
* Row - Focus 行 - 焦点：当用户将焦点放在一行上时应用的外观。
* Section Header 节标题：应用于“区段”小组件的“节标题”的外观。
* Widget 控件：应用于整个区段的外观。
* Row - Alternate 行 - 备用：应用于区段中每个备用偶数行的外观。
* Blocked UI 阻止的 UI：在正在进行的操作（例如，服务调用）完成之前阻止接口的外观。> 注意：阻止的 UI 仅适用于 SPA 平台。
* Pressed 压：指示段的行被按下或单击的外观。
* Refresh - Pull 刷新 - 拉取：拉动滚动条时小组件的外观。
* Refresh - Push 刷新 - 推送：按下滚动条时小组件的外观。

## Segment Properties 区段属性
### Master Data 主数据
仅当将小组件添加到“区段”小组件时，才会启用主数据。

### Row Template 行模板
指示在创建行和填充数据时要用于每行的通用模板。使用模板键设置数据时，可以在行级别覆盖此项。您可以使用下拉列表中的“新建”选项创建新模板，而无需转到“模板”部分。您还可以在区段中内联编辑模板。

### Section Header Template 章节标题模板
略

### Group Cells 组单元格
指定是否应使用圆角背景和边框对线段中的所有行进行分组。

### Retain Selection 保留所选内容
指定区段是否应保留用户导航出窗体并重新访问窗体时所做的选择。

### Full Screen Widget 全屏小部件
指定小组件是否应占据整个容器。如果您的区段具有大型数据集（超过 20 条记录，每条记录具有 15 个以上的小组件），则必须将该值设置为 true，以便更好地重用小组件和不同的滚动行为。

### Separator 分隔符
指定段是否应显示行之间的分隔符。

### Separator Thickness 分离器厚度
指定分隔符的粗细（以像素为单位）。

### Separator Color 分隔符颜色
指定“段”小组件的行之间的分隔符的颜色。

### Separator Transparency 分离器透明度
为分离器提供所需的透明度。

### Show Scrollbars 显示滚动条
指定段的滚动条是否必须始终可见。

### Orientation 取向
指定如何在区段内堆叠小组件。您可以将线段的方向设置为水平或垂直。

### View Type 视图类型
* 表：区段的行以列表形式显示在表中。
* 页面：区段的行显示在页面中，您需要滚动页面才能查看行。

### Layout Alignment 布局对齐
指定小组件的布局方向。

### Selection Behavior 选择行为
指定区段是支持单选还是多选。

* 默认值：表示区段不支持单选或多选。此选项允许您为区段定义 onRowClick 事件。
* 单选：指示在段中有多个选择时可以进行一个选择（行为类似于 RadioButtonGroup）。
* Multi Select：指示当您在区段中有多个选择时，您可以进行多个选择（行为类似于 CheckBoxGroup）。

### Enable Reordering 启用重新排序
对于 Volt MX Iris 版本 9.2 及更高版本，指定是启用还是禁用区段中的行重新排序。

### Autogrow Mode 自动增长模式
仅当段放置在 flex 容器中且“视图类型”设置为“表”时，此属性才适用。它指定在添加新内容时要增长的区段。

### Height 高度
以百分比为单位指定段的高度。该百分比参考了“高度参考”属性的值。

### Height Reference 高度参考
“段高百分比”是根据所选选项计算得出的。

### Dictionary 字典
指定是否必须启用词典以便于导航。

### Indicator 指示器

### Edit Style 编辑样式
指定启用区段编辑功能的方式。

* 图标：每行的左侧将显示一个图标。
* 滑动：当用户在每行上执行滑动手势时，每行的右侧将显示一个删除或插入按钮。是显示插入按钮还是删除按钮由使用 Segment 的 data 属性设置的 editmode 属性控制。
* 无：不应用特殊的编辑样式。

### Progress Indicator 进度指示器
指定是否显示进度指示器。

### 进度指示器颜色
将进度指示器的颜色指定为白色或灰色。

### Scroll Bounces 滚动反弹
指定滚动视图是否反弹过内容的边缘并再次返回。

### Search Criteria 搜索条件
指定启用搜索时要应用的搜索条件。

### Search By 搜索依据
指定放置在区段行内的小组件的标识符。将针对小部件中存在的内容执行搜索。

### Dock Section Header 码头部分标题
指定在滚动节内容时是否将节标题停靠在节的顶部。如果要滚动区段数据，则下一节标题将停靠在区段的顶部

### Swipe Config 滑动配置
您可以为细分小部件配置滑动功能。通过轻扫配置设置，用户可以在向左或向右轻扫行时关闭行或显示某些操作。

## Actions 行动
* onRowClick：当用户单击小组件的任何行时，将触发该操作。
* onPull：当小部件从顶部拉取时，将触发该操作。
* onPush：从底部推送小部件时触发操作。
* onReachingBegining：当滚动到达小组件的开头时，将触发该操作。
* onReachingEnd：当滚动到达小组件的末尾时，将触发该操作。
* onTouchStart：当用户触摸触摸表面时触发操作。此事件以异步方式发生。
* onTouchMove：当触摸在触摸表面上连续移动直到移动结束时，将触发该操作。此事件以异步方式发生。
* onTouchEnd：当用户触摸从触摸表面释放时触发操作。此事件以异步方式发生。
* onEditing：当用户编辑行时触发该操作。
