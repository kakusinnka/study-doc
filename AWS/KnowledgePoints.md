## IAM（Identity and Access Management） 和 Access Policy（访问策略） 
### **3. IAM 和 Access Policy 的对比**

| **对比维度**     | **IAM（Identity and Access Management）** | **Access Policy（访问策略）**           |
| ---------------- | ----------------------------------------- | --------------------------------------- |
| **管理对象**     | 用户、组、角色、服务                      | AWS 资源（如 S3 存储桶、DynamoDB 表等） |
| **应用范围**     | 全局（可跨多个资源）                      | 资源级别（应用于特定资源）              |
| **策略附加位置** | 附加到用户、组、角色或服务                | 直接附加到资源                          |
| **跨账户访问**   | 通过角色实现                              | 通过策略直接允许跨账户访问              |
| **控制粒度**     | 更适合管理用户权限，支持细粒度控制        | 更适合管理资源权限，支持资源共享        |
| **典型使用场景** | 管理用户、组、角色的权限                  | 控制特定资源的访问权限                  |
| **策略格式**     | JSON 格式                                 | JSON 格式                               |
| **与资源的关系** | 间接管理资源（通过用户或角色访问资源）    | 直接管理资源                            |

---

### **总结**
- **IAM** 是以用户、组和角色为中心的权限管理工具，用于控制“谁可以做什么”。
- **Access Policy** 是以资源为中心的权限管理工具，用于控制“资源可以被谁访问”。

## EC2 实例类型快速记忆
| 实例类型  | 名字含义             | 特性                       | 适用场景                     | 联想记忆                             |
| --------- | -------------------- | -------------------------- | ---------------------------- | ------------------------------------ |
| **T3**    | **通用型**           | 平衡的计算、内存和网络资源 | 轻量级应用、开发测试         | **"T" 代表通用，适合多种轻量任务**   |
| **I3**    | **IOPS 优化型**      | 高 IOPS，NVMe SSD 存储     | 数据库、高速存储访问         | **"I" 代表 IOPS，高速数据读写**      |
| **I8g**   | **I = IOPS**         | 存储优化，快速读写         | 数据库、高速缓存             | **I = Input/Output**                 |
| **R5**    | **内存优化型**       | 大量内存                   | 内存密集型应用、大数据分析   | **"R" 代表 RAM，内存需求高的任务**   |
| **R8g**   | **R = RAM**          | 内存优化，大内存           | 内存数据库、大数据分析       | **R = RAM**（内存任务）              |
| **Hpc7g** | **HPC = 高性能计算** | 高性能计算，科学模拟       | 工程模拟、科学计算           | **HPC = 科学计算**                   |
| **Trn1**  | **Trn = Training**   | 机器学习训练优化           | 深度学习训练（如 LLM）       | **Trn = Train**（AI训练）            |
| **G4**    | **GPU 优化型**       | 配备 NVIDIA GPU（如 T4）   | 机器学习、深度学习、图形处理 | **"G" 代表 GPU，用于计算密集型任务** |
| **G5**    | **G = Graphics**     | 图形处理优化，支持推理任务 | 图形渲染、视频处理、推理任务 | **G = Graphics**（图形任务）         |
| **P5**    | **P = Performance**  | 高性能 GPU，适合训练和计算 | 深度学习训练、高性能计算     | **P = Performance**（高性能）        |
| **Inf2**  | **Inf = Inference**  | 推理优化，低延迟高吞吐量   | 机器学习推理（如实时推荐）   | **Inf = Inference**（推理）          |

### 解释
- **G4 实例**：专为需要大量计算能力的任务设计，如机器学习和图形处理。
- **I3 实例**：适用于需要快速数据访问的场合，如数据库和日志处理。
- **R5 实例**：提供大量内存，适合需要高内存的应用程序。
- **T3 实例**：通用型，适合各种轻量级和通用计算任务。
## 以下是关于 **Customer Gateway**、**Internet Gateway**、**AWS Transit Gateway** 和 **AWS PrivateLink** 的对比表格：

| **名称**                | **用途**                                                             | **适用场景**                                                     | **是否支持限制互联网访问** | **特点**                                                                   |
| ----------------------- | -------------------------------------------------------------------- | ---------------------------------------------------------------- | -------------------------- | -------------------------------------------------------------------------- |
| **Customer Gateway**    | 用于在本地数据中心与AWS之间建立VPN连接。                             | 本地数据中心需要通过VPN访问AWS资源的场景。                       | 否                         | - 连接本地网络和AWS。<br>- 需要配置VPN。                                   |
| **Internet Gateway**    | 用于VPC与互联网之间的通信，允许VPC中的资源访问互联网或被互联网访问。 | 需要公开访问互联网的场景，例如托管公共Web应用程序。              | 否                         | - 提供VPC的出入互联网流量。<br>- 允许公网IP访问。                          |
| **AWS Transit Gateway** | 用于连接多个VPC和本地网络，集中管理网络流量。                        | 多个VPC之间通信，或本地数据中心与多个VPC通信的复杂网络结构场景。 | 否                         | - 支持大规模网络连接。<br>- 集中管理网络流量。                             |
| **AWS PrivateLink**     | 用于在VPC之间或VPC与AWS服务之间建立私有连接，避免通过互联网通信。    | 需要限制互联网访问，但仍需安全访问VPC内的服务或AWS服务的场景。   | 是                         | - 通过私有网络通信。<br>- 提高安全性和隐私性。<br>- 无需暴露服务到互联网。 |
