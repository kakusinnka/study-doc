### 总结
* “英雄之旅”应用程序显示具有详细信息视图的英雄列表。
* 用户可以选择一个英雄并查看该英雄的详细信息。
* 您使用 *ngFor 显示英雄列表。
* 您使用 *ngIf 有条件地包含或排除 HTML 块。
* 您使用 class 绑定切换 CSS 样式类。

## 创建特征组件
* 您创建了一个单独的、可重用的 HeroDetailComponent.
* 您使用了属性绑定来授予父级 HeroesComponent 对子项 HeroDetailComponent 的控制权。
* 您使用修饰器 @Input 使 hero 属性可供外部 HeroesComponent .

## 添加服务
* 通过 HeroService 服务类，重构了对数据的访问。
* 将 HeroService 服务注册到根级别，成为其服务的提供程序，以便可以在应用程序中的任何位置注入它。
* 使用了 Angular 依赖注入将服务注入到组件中。
* 为 服务的 getData 方法提供了异步签名。
* 您发现了 RxJS 库。
* 你用RxJS of() 返回一个可观察到的模拟英雄对象。
* 使用组件的 ngOnInit 生命周期挂钩调用 HeroService 的方法，而不是构造函数。
* 您创建了一个 MessageService 用于类之间的松散耦合通信。
* 注入到组件中的是 HeroService 使用另一个注入的服务 MessageService.

## 使用路由添加导航
* 您添加了 Angular 路由器以在不同组件之间导航
* 您将 AppComponent 变成了带有 <a> 链接和 <router-outlet> 的导航 shell
* 您在 AppRoutingModule 中配置了路由器
* 您定义了路由、重定向路由和参数化路由
* 您在锚元素中使用了 routerLink 指令
* 您将紧密耦合的主/详细视图重构为路由详细视图
* 您使用路由器链接参数导航到用户选择的英雄的详细视图
* 您与其他组件共享了 HeroService
