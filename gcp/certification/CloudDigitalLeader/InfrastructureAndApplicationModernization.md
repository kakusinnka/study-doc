# 使用 Google Cloud 实现基础架构和应用程序现代化
    许多传统企业使用遗留系统和应用程序，这些系统和应用程序通常很难达到满足现代客户期望所需的规模和速度。
    业务领导者和 IT 决策者必须不断地在维护遗留系统和投资创新产品和服务之间做出选择。
    本课程探讨了过时 IT 基础架构的挑战，以及企业如何使用云技术对其进行现代化改造。
    首先探索云中可用的不同计算选项以及每个选项的优势，然后再转向应用程序现代化和应用程序编程接口 (API)。
    还考虑了一系列可帮助企业更好地开发和管理其系统的 Google Cloud 解决方案，例如 Compute Engine、App Engine 和 Apigee。

## 目标
* 参考云技术描述基础架构和应用程序现代化的主要优势。
* 区分虚拟机、容器和 Kubernetes。
* 解释可帮助企业更好地管理其系统的 Google Cloud 解决方案，例如 App Engine 和 Google Kubernetes Engine。
* 解释 Google Cloud 解决方案如何支持应用现代化并简化 API 管理。

## 基础设施现代化
* 摆脱我们所谓的本地基础设施的第一步是托管。
* 虚拟机共享同一个计算机处理器、存储和网络资源池。虚拟机优化了可用资源的使用率，使企业能够以高效且可管理的方式在服务器上同时运行多个应用程序。
* 在基础架构级别外包您的 IT 需求称为基础设施即服务。公共云提供商（如 Google Cloud）提供多种服务来帮助您实现基础架构现代化。
* 如果您想要更多的托管服务，云服务提供商会提供一种称为平台即服务的东西。在这种情况下，您不必管理基础设施，而且对于某些服务，您只需为使用的部分付费。

## 了解云中的计算选项
* 可用于实现基础架构现代化的三个主要选项：虚拟机、容器化和无服务器计算。
* 虚拟化是一种资源优化形式，允许多个系统在同一硬件上运行。这些系统称为虚拟机或 VM。这意味着它们共享相同的计算机处理、存储和网络资源池。VM 使企业能够以高效且可管理的方式在服务器上同时运行多个应用程序。启用此功能的软件层称为管理程序。管理程序位于物理硬件之上，多个 VM 构建于其之上。这就像有多台计算机只使用一个硬件。虚拟机是基础设施现代化的第一个计算选项。
* 第二个是容器。 容器遵循与虚拟机相同的原则。它们提供隔离的环境来运行您的软件服务并从一个硬件优化资源。然而，它们的效率更高。虚拟机重新创建硬件的完整表示。相比之下，容器仅重新创建或虚拟化操作系统。这意味着它们只包含它们所支持的特定应用程序所需的确切内容。容器为开发人员和 IT 运营团队提供了一个更轻量级的单元来使用并提供一系列好处。与启动整个操作系统相比，它们启动速度更快，并且只使用一小部分内存。容器使开发人员能够创建与其他资源隔离的可预测环境。容器可提高敏捷性、增强安全性、优化资源并简化云中应用程序的管理。
* 随着 IT 基础架构设置变得越来越复杂，企业需要一种方法来管理其服务和机器。Kubernetes 是一个开源集群管理系统，可提供自动化容器编排。换句话说，Kubernetes 为您简化了机器和服务的管理。这提高了应用程序的可靠性，并减少了您需要花费在开发和运营上的时间和资源。
* 最后，第三个计算选项是无服务器计算。无服务器计算并不意味着没有服务器。无服务器计算意味着计算能力等资源会根据需要在幕后自动配置。这意味着企业无需为计算能力付费，除非他们实际运行查询或应用程序。简而言之，无服务器意味着企业为他们想要的任何功能提供代码，公共云提供商做其他一切。无服务器计算解决方案通常被称为功能即服务。一些功能是对特定事件的响应，例如文件上传到您的云存储或更改您的数据库记录。您编写定义对这些事件的响应的代码，而您的云提供商将完成其他所有工作。

## 混合云和多云
* 私有云是组织在自己的数据中心拥有虚拟化服务器以创建自己的私有本地环境的地方。
* 混合云是指组织使用本地或私有云基础设施和公共云服务的某种组合。
* 多云是指组织使用多个公共云提供商作为其架构的一部分。

## 谷歌云计算解决方案
* 基于 VM 的计算选项，包括 Compute Engine、Google Cloud VMware Engine 和 Bare Metal。
  * Compute Engine 是一种计算和托管服务，可让您在 Google 基础设施上创建和运行虚拟机。
  * Google Cloud VMware Engine 是一项完全托管的服务，可让您在 Google Cloud 中运行 VMware 平台。
  * Bare Metal 是指没有安装操作系统或虚拟化软件的物理服务器。与传统的虚拟化环境不同，裸金属服务器允许直接访问服务器的硬件资源，而无需在操作系统或虚拟化层之上运行。
* Google Kubernetes Engine（GKE），这是一个基于容器的计算选项。GKE 提供了一个托管环境，用于使用 Google 基础设施部署、管理和扩展容器化应用程序。
* 三种无服务器计算解决方案，即 App Engine、Cloud Functions 和 Cloud Run。
  * Google App Engine 是一个平台即服务和云计算平台，用于开发和托管 Web 应用程序。
  * Cloud Run 允许开发人员使用他们最喜欢的依赖项和工具以他们最喜欢的编程语言构建应用程序，并在几秒钟内部署它们。
  * Cloud Functions 是用于构建和连接云服务的无服务器执行环境。它提供可扩展、即用即付的功能作为服务，以零服务器管理运行您的代码。
  * Google App Engine 适用于快速部署和扩展应用程序的开发人员，具有更高级的抽象和自动化。Cloud Run 适合那些希望在容器级别上更灵活和自由度更高的开发人员，可以使用任何编程语言和框架。Cloud Functions 则专注于事件驱动的无服务器函数，用于编写轻量级的函数以响应特定的事件触发器。

# 使用 Google Cloud 实现应用程序现代化
    本模块探讨企业如何实现现有应用程序的现代化并在云中构建新的应用程序。

## 简介 
* 我将探讨企业如何实现现有应用程序的现代化并在云中构建新的应用程序。特别是，我将为想要对其应用程序进行现代化改造的企业介绍五种常见的更改模式。
* 我将考虑企业面临的一些关键应用程序开发挑战。
* 然后我将重点介绍两个 Google Cloud 解决方案，即 Google Kubernetes Engine 和 App Engine。

## 云变化模式
* Google Cloud 确定了企业在想要实现应用程序现代化时可以采用的五种常见模式。企业可以先将应用程序迁移到云中，然后再进行更改，或者他们可以在迁移之前更改他们的应用程序，或者他们可以在绿地中进行发明，或者在棕地中进行发明，或者他们可以不做任何更改地移动他们的应用程序。
* 如果组织希望采取相对保守的方法通过云实现应用程序现代化，他们可能会先采取行动，然后再改变方法。
* 如果组织希望采取更积极的方法来实现应用程序现代化，他们可以首先重新架构应用程序，使它们在迁移之前更加适合云计算。
* 当我们谈论绿地战略时，我们谈论的是在云中构建全新的基础设施和应用程序。
* 棕地战略是在云环境中发明一种新应用程序，以取代保留在本地的现有遗留应用程序。遗留应用程序仅在构建新应用程序后才退役。
* 最后，值得注意的是，构建云原生应用程序并不适合所有场景。对于某些用例，利用云来实现基础设施层的现代化就是非常有效的。

## 应用引擎
* App Engine 是一个用于构建可扩展的 Web 应用程序和移动后端的平台。

## 旧系统挑战
* 遗留系统的开发并不是为了支持云、物联网或移动应用程序等现代技术的实施和采用。
* 其次，它们是为批量或按特定时间间隔共享数据而开发的。

## 如何使遗留系统现代化
* 如何使用它来实现基础设施和 API 的现代化，将其作为连接不同应用程序并使信息能够在系统之间流动的软件？最终，API 可实现系统之间的集成，以便企业能够释放价值并创建新服务。他们通过以保护遗留系统完整性并实现对底层数据的安全和管理访问的方式公开数据来实现这一点。这使得拥有旧系统的组织能够适应现代业务需求，更重要的是，能够快速采用新技术和平台。

## 使用 API 创造新的业务价值
* 当组织开始思考并构建 API 优先架构时，他们就可以构建新的数字生态系统并创造新的业务价值。
* API 不仅是一项长期存在的集成技术，而且是数字化转型的基本构建模块。

## Apigee
* Apigee是一种领先的 API 管理平台，它为企业提供了强大的工具和功能，用于设计、部署和管理应用程序接口（API）。它使企业能够构建安全、可靠和可扩展的 API，简化 API 开发和管理过程，并为开发人员和业务团队提供了丰富的功能和洞察力。

# 概括

## 总结
* 企业需要提供无缝的数字体验，才能在云时代保持相关性。
* 旧系统和应用程序无法提供这些体验所需的实时速度和规模。
* IT 基础设施现代化本质上是将单一系统和应用程序分解为使用共享资源池的最小单元的过程。因此，当出现新的功能请求或需要更改时，每个单元都可以轻松地大规模更新和部署。
* 应用程序编程接口或 API 通常位于遗留系统和现代应用程序之间，使以前无法访问的信息能够在它们之间流动。
